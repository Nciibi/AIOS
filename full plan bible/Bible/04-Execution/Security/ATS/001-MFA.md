## Purpose

Multi-Factor Authentication provides the highest assurance level for identity verification. MFA is required for Security Council actions, constitutional overrides, and critical operations.

## Factor Combinations

| Combination | Security Level | Typical Use |
|------------|---------------|-------------|
| Password + TOTP | High | User administrative actions |
| Cryptographic key + Biometric | Very High | Engine-to-Engine critical operations |
| Password + Hardware key (FIDO2) | Very High | Organization-level administrative actions |
| Cryptographic key + SMS code | Medium | Fallback when TOTP unavailable |
| Three-factor (password + TOTP + hardware key) | Maximum | Security Council actions, constitutional amendment operations |

## MFA Flow

```
User authenticates with primary factor (password)
  → If MFA required for this operation:
    → Prompt for secondary factor
    → User provides secondary factor (TOTP, biometric, etc.)
    → Verify secondary factor against registered secret
    → If valid: issue MFA-enhanced token
    → Cache MFA session for 15 minutes (configurable)
  → If MFA not required:
    → Issue standard token
```

## Session Cache

Once MFA is completed, the secondary factor verification is cached for 15 minutes by default. During this window, the entity may perform other MFA-required operations without re-entering the secondary factor. Cache parameters:
- Default TTL: 15 minutes
- Maximum TTL: 60 minutes (configurable per Organization)
- Cache invalidation: on any identity lifecycle change, token revocation, or explicit logout
- Cache scope: per entity + per Security Council instance (not shared across instances without explicit policy)

## MFA Enrollment

| Step | Description |
|------|-------------|
| 1 | Entity authenticates with primary factor |
| 2 | Entity registers secondary factor (TOTP secret, biometric template, or hardware key) |
| 3 | Secondary factor is verified (challenge-response) |
| 4 | Secondary factor hash/verification data stored in secure store |
| 5 | Entity receives confirmation of successful MFA enrollment |

## Fallback

If the entity's primary MFA factor is unavailable:
- Pre-generated recovery codes (10 codes, single-use)
- Security Council override (requires 2 Security Council members to authorize)
- Time-based escalation: after 24 hours without MFA, automatic escalation to Security Council
