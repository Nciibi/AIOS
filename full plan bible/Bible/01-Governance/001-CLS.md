## Purpose

The Constitutional Lifecycle Service governs the Constitution itself — how it is created, versioned, amended, interpreted, and enforced. CLS ensures that the Constitution remains the supreme authority and that all changes to it are traceable, evidenced, and authorised.

## Constitution Structure

The Constitution consists of:
- **Laws**: The 10 immutable constitutional laws (Physics/000-Laws.md). These change only through the constitutional amendment process.
- **Physics Invariants**: The normative invariants that implement the Laws. These change only when the corresponding Law changes.
- **Bible Specifications**: The detailed specifications that implement Physics invariants. These change through the RFC process.
- **Interpretations**: Authoritative interpretations of the Constitution by the Security Council. These are advisory and may be superseded by RFC.

## Constitution Lifecycle

```
Draft → Ratified → Active → Amended → Superseded
```

| State | Description | Can Be Modified? |
|-------|-------------|------------------|
| **Draft** | Constitution is being written. Not yet in force. | Yes |
| **Ratified** | Constitution has been approved. In force. | No (amendment required) |
| **Active** | Constitution is the current governing document. | No (amendment required) |
| **Amended** | A specific article or section has been modified via RFC. The Constitution remains Active. | No (further amendment required) |
| **Superseded** | Constitution has been replaced by a newer version. | No |

## Amendment Process

### Constitutional Amendment (Laws Change)

**Triggers**: A proposed change to one or more of the 10 Laws. This is the highest-impact governance action.

**Authorization Required**:
1. RFC submitted by any constitutional entity
2. Sou approval (majority vote)
3. Security Council approval (unanimous consensus)
4. Evidence of need (Academy analysis, incident report, or constitutional review)

**Process**:
```
1. RFC-NNNN submitted to CRP (03-Governance/003-CRP.md)
2. CRP routes to CLS for constitutional amendment classification
3. Evidence package assembled (why the Law must change)
4. Sou review and vote (majority)
5. Security Council review and vote (unanimous)
6. If approved: Law version incremented (e.g., v1.0 → v2.0)
7. Physics documents updated to match
8. Bible documents updated to match
9. Amendment Event recorded in Event Store
10. All entities notified of new Constitution version
```

### Bible Amendment (Specification Change)

Bible documents change through the standard RFC process (see 003-CRP.md). No Law change is required. The RFC must cite which Law the specification implements.

### Clerical Correction

**Triggers**: Typographical error, formatting fix, clarification without semantic change.

**Authorization Required**: Security Council approval (simple majority).

**Process**: RFC with `type: clerical` classification. Skip Sou review.

## Enforcement

CLS enforces the Constitution through:
1. **Constitutional Review**: Every RFC is checked against the Constitution before approval. An RFC that violates the Constitution is automatically rejected.
2. **Runtime Enforcement**: The Security Council's verification pipeline checks every action against constitutional policies (Stage 4 of the pipeline).
3. **Audit**: Every action that potentially violates the Constitution is flagged for Security Council review.
4. **Constitutional Challenge**: Any entity may challenge any action as unconstitutional. The challenge is reviewed by the Security Council within 24 hours.
