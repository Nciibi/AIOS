## Purpose

Autonomous Knowledge Management governs how the Academy learns, stores, verifies, and distributes knowledge. AKM ensures that knowledge is:
- **Constitutional** — consistent with the Constitution and Physics invariants
- **Evidenced** — traceable to source Events
- **Verifiable** — reproducible from source evidence
- **Privacy-preserving** — respects organizational boundaries
- **Quality-controlled** — reviewed and validated before acceptance

## Knowledge Governance Model

Academy knowledge exists within a governance framework that describes the conceptual flow of knowledge:

```
Evidence → Analysis → Validation → Acceptance → Distribution → Review
```

This conceptual flow is distinct from the formal knowledge lifecycle (below). The governance model describes the high-level stages; the knowledge lifecycle defines the exact state machine that the Academy implements.

| Governance Stage | Corresponds to Lifecycle State(s) |
|-----------------|----------------------------------|
| Evidence | Generated (knowledge produced from evidence) |
| Analysis | Generated → Proposed |
| Validation | Validated |
| Acceptance | Accepted |
| Distribution | Published |
| Review | Deprecated → Archived (continuous improvement) |

## Knowledge Types

| Type | Description | Validation Required | Access |
|------|-------------|-------------------|--------|
| **Operational** | How to perform specific tasks (runbooks, playbooks) | Medium | Entity + parent Organization |
| **Domain** | Domain-specific expertise (coding, trading, security) | High | Domain members |
| **Constitutional** | Constitution interpretation, governance knowledge | Highest | All entities |
| **Strategic** | Planning, resource optimisation, decision support | High | Sou + Security Council |
| **Experimental** | Tentative knowledge under evaluation | Low | Researchers (explicit consent) |

## Knowledge Lifecycle

```
Generated → Proposed → Validated → Accepted → Published → Deprecated → Archived
```

| State | Description | Authorized Mutator |
|-------|-------------|-------------------|
| **Generated** | Knowledge is produced by an entity's learning process | Academy Engine |
| **Proposed** | Knowledge is submitted for validation | Source entity |
| **Validated** | Knowledge passes quality and constitutional checks | Knowledge Validator |
| **Accepted** | Knowledge is accepted into the Academy | Knowledge Reviewer |
| **Published** | Knowledge is distributed to consumers | Academy Distribution |
| **Deprecated** | Knowledge is superseded or invalidated | Knowledge Reviewer or Security Council |
| **Archived** | Knowledge is preserved for audit, not actively distributed | Academy (automatic) |

## Validation Rules

### Constitutional Validation (All Knowledge)

Every knowledge artifact must pass:
1. **Constitutional consistency**: Does not violate any Law
2. **Evidence provenance**: Traceable to specific Events
3. **Non-contradiction**: Does not contradict accepted knowledge (unless superseding it)
4. **Privacy compliance**: Does not expose evidence beyond authorized boundaries

### Quality Validation

| Rule | Applies To | Check |
|------|-----------|-------|
| Reproducible | Operational, Domain | Knowledge can be reproduced from source evidence |
| Complete | All | Knowledge has sufficient context for its purpose |
| Current | All | Knowledge is not superseded by a newer artifact |
| Non-duplicative | All | Knowledge is not a duplicate of existing accepted knowledge |
| Deterministic | Operational | Knowledge produces deterministic outcomes (R9) |

## Access Control

Knowledge access is governed by:
1. **Organizational boundary**: Knowledge produced by Organization A is not accessible to Organization B unless explicitly shared
2. **Autonomy level**: Higher autonomy entities may access more knowledge
3. **Knowledge type**: Constitutional knowledge is universally accessible; operational knowledge is scoped
4. **Entity type**: Some knowledge is restricted to specific entity types (Sou, Security Council)

## AKM Events

| Event Type | Produced When | Fields |
|-----------|--------------|--------|
| `AKM.KnowledgeProposed` | Knowledge is submitted for acceptance | knowledge_id, source_entity_id, knowledge_type, evidence_hash |
| `AKM.KnowledgeValidated` | Knowledge passes validation | knowledge_id, validator_id, validation_result |
| `AKM.KnowledgeAccepted` | Knowledge is accepted into Academy | knowledge_id, reviewer_id, acceptance_timestamp |
| `AKM.KnowledgePublished` | Knowledge is distributed to consumers | knowledge_id, published_to |
| `AKM.KnowledgeDeprecated` | Knowledge is marked as deprecated | knowledge_id, superseded_by, reason |
| `AKM.KnowledgeArchived` | Knowledge is archived | knowledge_id, retention_period |
