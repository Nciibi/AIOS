## Purpose

The Decision Gateway Process governs how constitutional decisions flow from Sou (the will engine) to implementation. DGP ensures that every decision with constitutional impact is routed through the correct governance process, evidenced, and traceable to its source.

## Decision Types

| Type | Description | Requires RFC? | Approving Authority |
|------|-------------|---------------|---------------------|
| **Strategic** | High-level direction, priority, resource allocation | Yes | Security Council |
| **Architectural** | System structure, module boundaries, interface contracts | Yes | Security Council + Architecture Review |
| **Constitutional** | Law interpretation, amendment, clarification | Yes (constitutional) | Sou + Security Council |
| **Operational** | Day-to-day decisions within existing authority | No | Entity itself |
| **Emergency** | Immediate action required to prevent harm | No (retrospective RFC required) | Security Council |
| **Experimental** | Temporary deviation for learning purposes | No (time-boxed, supervised) | Security Council |

## Decision Lifecycle

```
Proposed вҶ’ Assessed вҶ’ Routed вҶ’ Reviewed вҶ’ Approved вҶ’ Implemented вҶ’ Verified
```

| State | Description | Owner |
|-------|-------------|-------|
| **Proposed** | Sou or any entity submits a decision proposal | Proposer |
| **Assessed** | DGP classifies the decision type and impact | DGP Engine |
| **Routed** | Decision is sent to the correct gateway | DGP Engine |
| **Reviewed** | Gateway reviews evidence, impact, and constitutional alignment | Gateway (ADG, CLS, etc.) |
| **Approved** | Security Council gives final approval | Security Council |
| **Implemented** | Decision is translated into RFC or direct action | Implementation team |
| **Verified** | Implementation is verified against the decision | Security Council |

## Decision Evidence

Every decision must be supported by evidence:
- **Strategic decisions**: Academy analysis, resource data, mission outcomes
- **Architectural decisions**: ADG template (005-ADG.md), impact analysis, dependency graphs
- **Constitutional decisions**: Legal reasoning, precedent, constitutional text
- **Operational decisions**: Entity's own evidence chain
- **Emergency decisions**: Incident report, post-hoc evidence collection

## DGP Events

| Event Type | Produced When | Fields |
|-----------|--------------|--------|
| `DGP.DecisionProposed` | Sou or entity submits proposal | decision_id, proposer_id, decision_type, summary |
| `DGP.DecisionAssessed` | DGP classifies the decision | decision_id, classification, impact_level, routed_to |
| `DGP.DecisionApproved` | Security Council approves | decision_id, approved_by, conditions |
| `DGP.DecisionRejected` | Gateway or Council rejects | decision_id, rejected_by, reason |
| `DGP.DecisionImplemented` | Implementation completes | decision_id, rfc_id (if applicable), verification_hash |
