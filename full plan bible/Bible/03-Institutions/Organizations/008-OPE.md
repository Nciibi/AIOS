## Purpose

The Organization Performance Evaluator (OPE) measures, reports, and improves Organization performance. OPE collects metrics from across the system â€” Mission completion rates, resource efficiency, Worker satisfaction, policy compliance, knowledge contribution â€” and produces actionable reports for Org leadership, the Security Council, Sou, the Academy, and DTS.

### Primary Metrics

| Metric | Description | Target | Calculation | Data Source |
|--------|-------------|--------|-------------|-------------|

## OPE Data Sources and Integration

OPE aggregates data from multiple sources to compute metrics:

| Metric | Primary Source | Secondary Source | Refresh Rate |
|--------|--------------|-----------------|--------------|
| Mission Completion Rate | LMS (Mission lifecycle Events) | Sou Mission evaluations | Real-time |
| Average Mission Duration | LMS (timestamps per state transition) | None | Real-time |
| Resource Efficiency | ROS (Budget consumption logs) | OHS (resource utilization) | Near real-time (5 min) |
| Engine Satisfaction | WHS (Worker health scores) | EEE (employment records) | Hourly |
| Policy Compliance | Security Council (verification Events) | OHS (compliance scan results) | Real-time |
| Knowledge Contribution | Academy (knowledge artifact registry) | None | Daily |

### Metric Computation Pipeline

```
1. Raw data collected from source systems via ACF event streams
2. Data normalized to standard metric format (value, unit, timestamp)
3. Metric computed using formula (see Primary Metrics table)
4. Grading applied (A-F scale based on target comparison)
5. Trend calculated (improving/stable/declining over 30-day window)
6. Metric stored in OPE metric store
7. If metric crosses warning/critical threshold â†’ alert generated
8. If all metrics computed â†’ health score calculated
9. Health score â†’ health state mapped
```

## Cross-Cutting Concerns

### Security

Performance reports contain sensitive operational data. Access is restricted to Org Admin (own Org) and Security Council (any Org). Cross-Org comparison is Security Council only. Metric targets changes require Admin authorization. (Physics/008-Security.md)

### Evidence

Every OPE operation produces an Event. Performance snapshots, reports, and improvement actions are recorded in the Event Store. Metric history is fully auditable. (PHI-008)

### Lifecycle

OPE operates for Organizations in Active state. Suspended Organizations do not generate reports. Dissolved Organizations have final performance reports archived for audit. Performance metrics from the Organization's lifecycle inform lifecycle decisions (e.g., health-based suspension). (PHI-006)

### Capability Bounds

OPE is a measurement and recommendation system â€” it does not directly modify Organization structure, budgets, or personnel. Improvement actions are suggestions; execution requires Admin or Security Council authorization. (Physics/007-Capabilities.md)

### Communication

Performance reports are delivered via ACF. Metric alerts are published to ACF event streams. Improvement action notifications are routed to Org leadership. (Law 3 â€” Communication)
