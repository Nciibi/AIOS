## Purpose

The Organization Interaction Service (OIS) enables cross-Organization communication and cooperation. Organizations are independent constitutional entities that may need to request resources from each other, propose joint Missions, share knowledge (under CPR-010), or delegate authority. OIS provides the constitutional framework for these interactions, ensuring all cross-Organization messages are authenticated, authorized, evidenced, and routed through ACF.

## Cross-Org Authorization

Every cross-Org message requires:
1. **Authentication** â€” both Organizations must have verified identities (IDS)
2. **Authorization** â€” the requesting Org must have authority for the specific request type
3. **Scope validation** â€” the request must be within both Orgs' capability bounds
4. **Evidence trail** â€” the complete interaction is recorded as Events

## Knowledge Sharing (CPR-010)

Under CPR-010 (Evidence Privacy), evidence is private to the Organization that produced it. Knowledge sharing through OIS requires:

1. Explicit consent from the sharing Organization's Admin
2. Anonymization of sensitive data (if required by policy)
3. A clear purpose for sharing (Academy learning, cross-Org optimization)
4. The receiving Organization may not redistribute without consent

| Knowledge Type | Default Sharing | Requires Consent? |
|---------------|----------------|-------------------|
| Operational Metrics | Org-private | Yes |
| Mission Outcomes | Org-private | Yes |
| Anonymized Patterns | Shareable within security bounds | No (anonymized by policy) |
| Constitutional Interpretations | Shareable | No (constitutional knowledge is public) |

## Cross-Cutting Concerns

### Security

All cross-Org communication requires authentication and authorization at both ends. Cross-branch requests require Security Council authorization. Broadcast is restricted to Security Council only. All denials are recorded as Security Events. (Physics/008-Security.md)

### Evidence

Every OIS message produces an Event. Complete interaction histories are preserved in the Event Store. Knowledge sharing records include provenance and anonymization level. (PHI-008)

### Lifecycle

OIS operates only between Organizations in Active state. Suspended Organizations cannot initiate or respond to cross-Org requests. Dissolved Organizations are removed from OIS routing. (PHI-006)

### Capability Bounds

Cross-Org requests are bounded by both Organizations' capability scopes. An Organization cannot request resources or capabilities beyond its constitutional bounds. Joint Mission proposals must be within both Orgs' Mission domains. (Physics/007-Capabilities.md)

### Communication

All OIS messages flow through ACF â€” no direct inter-Org communication outside ACF. OIS is built on ACF's routing, authentication, and messaging capabilities. OIS adds the Organization-level interaction semantics on top of ACF. (Physics/009-Interaction.md)
