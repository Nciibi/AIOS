## Purpose

Organization Governance (ORG) defines how Organizations make decisions. Every Organization has a governance policy that specifies who may make which decisions, what evidence is required, whether a vote is needed, and how disputes are escalated. Governance ensures that Organizations act constitutionally, transparently, and accountably.

## Governance Model

Each Organization has a governance policy that maps decision types to authorizing roles, evidence requirements, and escalation paths. The policy is established at creation (via Genome) and may be amended by the Organization's Admin within constitutional bounds.

```
GovernancePolicy {
    org_id: UUID
    decision_rules: DecisionRule[]
    default_escalation: EscalationPath
    voting_config: VotingConfig
    amendment_policy: AmendmentRule
}
```

### Decision Types

| Type | Description | Examples |
|------|-------------|----------|
| **Operational** | Day-to-day decisions within existing authority | Assign Workers to Missions, approve routine resource requests |
| **Tactical** | Medium-term decisions affecting department or team scope | Adjust department budgets, create sub-teams, modify operational policies |
| **Strategic** | High-level decisions affecting Organization direction | Change governance policy, approve major Missions, restructure departments |
| **Constitutional** | Decisions affecting constitutional compliance | Dissolve Organization, amend genome, change role structure |

## Decision Authority Matrix

| Decision Type | Operational | Tactical | Strategic | Constitutional |
|--------------|-------------|----------|-----------|----------------|
| **Member** | Yes (within assigned scope) | No | No | No |
| **Manager** | Yes (department scope) | Yes (department scope) | Suggest only | No |
| **Admin** | Yes (Org-wide) | Yes (Org-wide) | Yes (Org-wide) | Propose only |
| **Council** | No | No | Yes (approve strategic) | Yes (approve constitutional) |

### Authority Levels

| Level | Entity | Scope | Can Override |
|-------|--------|-------|-------------|
| L1 ├втВмтАЭ Execute | Member | Assigned tasks | No |
| L2 ├втВмтАЭ Decide | Manager | Department | Admin |
| L3 ├втВмтАЭ Authorize | Admin | Organization | Security Council |
| L4 ├втВмтАЭ Approve | Security Council | Multi-Org/Constitutional | AI Council |

## Governance Policy Structure

```
DecisionRule {
    decision_type: DecisionType    // operational | tactical | strategic | constitutional
    authorizing_role: Role         // member | manager | admin | council
    requires_evidence: bool        // Must evidence be attached?
    requires_vote: VoteConfig?     // Voting rules if required
    escalation_path: EscalationStep[]  // Escalation if decision cannot be made
    auto_approve: bool             // Auto-approve if conditions met?
    max_duration: Duration         // Max time to resolve this decision
}
```

### Default Governance Policy

| Decision | Authorizing Role | Requires Evidence? | Requires Vote? | Escalation Path |
|----------|-----------------|-------------------|----------------|-----------------|
| Assign Mission | Manager | Yes | No | ├втАатАЩ Admin |
| Approve Budget | Admin | Yes (resource plan) | No | ├втАатАЩ Security Council |
| Create Department | Admin | Yes (rationale, impact) | Manager vote (simple majority) | ├втАатАЩ Security Council |
| Modify Governance Policy | Admin | Yes (constitutional check) | Admin vote (supermajority 2/3) | ├втАатАЩ Security Council |
| Dissolve Organization | Security Council + Sou | Yes (dissolution rationale) | Admin vote (unanimous) | ├втАатАЩ AI Council |
| Change Role Assignment | Authorizing role per role model | Yes (role change evidence) | No | ├втАатАЩ Next higher role |
| Approve Cross-Org Agreement | Admin | Yes (agreement terms) | Manager vote (simple majority) | ├втАатАЩ Security Council |

## Voting Rules

| Rule | Threshold | Description | Used For |
|------|-----------|-------------|----------|
| **Simple Majority** | >50% of votes | More yes than no | Operational decisions, department creations |
| **Supermajority** | ├втА░┬е2/3 of votes | Two-thirds majority | Policy changes, strategic decisions |
| **Unanimous** | 100% of votes | All must agree | Constitutional changes, dissolution |

### Voting Configuration

```
VotingConfig {
    rule: VotingRule               // simple_majority | supermajority | unanimous
    eligible_voters: Role[]        // Which roles may vote
    quorum: Float                  // Minimum % of eligible voters required (default: 50%)
    voting_period: Duration        // How long voting is open (default: 48h)
    abstention_default: bool       // Treat abstention as "no" (default: false)
    vote_visibility: Visibility    // public | anonymous | restricted
}
```

### Voting Process

```
1. Decision identified ├втАатАЩ Decision proposal created
2. Governance policy checked ├втАатАЩ authorizing role identified
3. If vote required ├втАатАЩ VotingConfig applied
4. Proposal distributed to eligible voters via ACF
5. Voting period opens (configurable per decision type)
6. Votes collected ├втВмтАЭ each vote produces an Event
7. At period end: quorum check ├втАатАЩ threshold check
8. If quorum not met: escalate to next authority
9. If threshold met: decision approved ├втАатАЩ action executed
10. If threshold not met: decision rejected ├втАатАЩ escalate if applicable
```

## Governance Events

| Event Type | Produced When | Fields |
|-----------|--------------|--------|
| `ORG.DecisionProposed` | Decision is formally proposed | decision_id, org_id, decision_type, proposer, summary |
| `ORG.DecisionAuthorized` | Authorizing role approves operational/tactical decision | decision_id, authorized_by, evidence_ref |
| `ORG.VoteInitiated` | Vote is opened for a decision | decision_id, voting_rule, eligible_voters, deadline |
| `ORG.VoteCast` | Voter casts a ballot | decision_id, voter_id, vote (yes/no/abstain) |
| `ORG.VoteCompleted` | Vote concludes with result | decision_id, outcome, vote_counts, turnout_pct |
| `ORG.DecisionApproved` | Decision is approved (via auto, role, or vote) | decision_id, approval_method, effective_at |
| `ORG.DecisionRejected` | Decision is rejected | decision_id, reason, rejected_by |
| `ORG.DecisionEscalated` | Decision is escalated to higher authority | decision_id, from_role, to_role, reason |
| `ORG.PolicyAmended` | Governance policy is modified | org_id, amended_rules, old_hash, new_hash |

## Cross-Cutting Concerns

### Security

Governance decisions are security-sensitive. Strategic and constitutional decisions require Security Council oversight. All decisions and votes produce Events for audit. Escalation paths ensure that no decision is stuck indefinitely. (Physics/008-Security.md)

### Evidence

Every governance action ├втВмтАЭ proposal, authorization, vote, escalation, approval, rejection ├втВмтАЭ produces an Event. Decision histories are fully auditable. Evidence requirements ensure decisions are grounded in constitutional facts. (PHI-008)

### Lifecycle

Governance decisions are lifecycle-state-dependent. Suspended Organizations cannot make governance decisions. Dissolved Organizations have no governance authority. Active Organizations have full governance capability. (PHI-006)

### Capability Bounds

Governance authority is bounded by role. No role may exercise authority beyond its defined scope. Constitutional decisions require Security Council + Sou approval ├втВмтАЭ no single entity may unilaterally modify constitutional structure. (Physics/007-Capabilities.md)

### Communication

All governance communications flow through ACF. Vote notifications, decision proposals, and escalation messages are routed through ACF event streams. Vote visibility is configurable per governance policy. (Law 3 ├втВмтАЭ Communication)
