## Purpose

The Organization Directory Service (ODS) is the query interface for the Organization tree. It provides lookup, hierarchy resolution, and search capabilities for all Organizations in AIOS. ODS is the system that answers "What Organizations exist?", "What is the structure of this Organization?", "Where does this Organization sit in the hierarchy?" ODS supports caching for query performance and enforces authorization boundaries for cross-branch visibility.

## ODS Operations

### resolveOrg

```
Input:  org_id
Process: lookup Organization by ID in Directory â†’ return record
Output: Organization (full record including parent, children, type, state)
Authorization: Member of target Organization or ancestor
Error: ODS_001 if not found, ODS_002 if unauthorized
Event: ODS.OrgResolved
```

### findOrg

```
Input:  query { name?, type?, state?, parent_id?, metadata_filters? }
Process: search Directory with filters â†’ return matching Organizations
Output: Organization[]
Authorization: Scoped to requester's hierarchy branch
Event: ODS.OrgSearchPerformed
```

### getAncestors

```
Input:  org_id
Process: traverse parent chain from org_id to root â†’ return list
Output: Organization[] (ordered from direct parent to root)
Authorization: Member of target Organization or any ancestor
Event: ODS.AncestorsQueried
```

### getDescendants

```
Input:  org_id, depth? (default: full subtree), type_filter?
Process: traverse children recursively up to depth â†’ return list
Output: Organization[] (breadth-first, all descendants)
Authorization: Member of target Organization
Event: ODS.DescendantsQueried
```

### getSiblings

```
Input:  org_id
Process: find parent â†’ list all children of parent excluding org_id â†’ return
Output: Organization[]
Authorization: Member of target Organization or parent
Event: ODS.SiblingsQueried
```

### getPath

```
Input:  org_id
Process: traverse parent chain from root to org_id â†’ return ordered path
Output: Organization[] (ordered from root to target)
Authorization: Member of target Organization or any ancestor
Event: ODS.PathQueried
```

## Caching

ODS maintains a cache of the Organization tree to serve queries with low latency.

| Cache Property | Value |
|---------------|-------|
| Cache type | Distributed read cache (all ODS replicas) |
| TTL (positive) | 5 minutes |
| TTL (negative â€” not found) | 1 minute |
| Invalidation trigger | Any structural change (create, move, dissolve) |
| Invalidation scope | Target Organization + all ancestors (path affected) |
| Cache warming | On startup: pre-load all Active Organizations |

### Cache Invalidation Rules

| Operation | Invalidated |
|-----------|-------------|
| Organization created | Parent's children list, hierarchy paths containing parent |
| Organization moved | Old parent's children, new parent's children, full path of moved Org |
| Organization dissolved | Parent's children list, all descendant paths |
| Organization metadata updated | The Organization's record only |
| Organization state changed | The Organization's record (lifecycle_state field) |

## Authorization Model

| Query | Default Visibility | Cross-Branch Visibility |
|-------|-------------------|------------------------|
| resolveOrg | Own Org + ancestors + descendants | Requires Security Council authorization |
| findOrg | Own hierarchy branch | Requires Security Council authorization |
| getAncestors | Any (ancestors are always visible to descendants) | N/A |
| getDescendants | Own Org only | Requires Org Admin or Security Council |
| getSiblings | Own Org + parent | Requires parent Admin for cross-branch |
| getPath | Own Org + ancestors | N/A |

### Cross-Branch Authorization

Cross-branch queries (resolving an Organization outside the requester's hierarchy branch) require:
1. Explicit Security Council authorization token
2. A legitimate operational need (joint Mission, resource transfer, audit)
3. The authorization is logged as a Security Event
4. The authorization has an expiration (default: 24 hours)

## Cross-Cutting Concerns

### Security

ODS enforces authorization boundaries. Organization tree visibility is scoped to the requester's hierarchy branch. Cross-branch access requires Security Council authorization. All queries are logged as Events for audit. (Physics/008-Security.md)

### Evidence

Every ODS query produces an Event. Query patterns are analyzable for operational intelligence. Cross-branch access authorizations and denials are recorded as Security Events. (PHI-008)

### Lifecycle

ODS only returns Organizations in non-terminal lifecycle states (Created, Verified, Active, Suspended, Restored). Dissolved and Archived Organizations require explicit filter override with Security Council authorization. (PHI-006)

### Capability Bounds

ODS is a read-only service â€” it does not modify Organization records. All write operations go through OOM and OSYS. ODS capabilities are limited to query and cache management. (Physics/007-Capabilities.md)

### Communication

All ODS queries and responses flow through ACF. Cache invalidation messages are broadcast on ACF event streams. ODS replicas communicate through ACF for cache synchronization. (Law 3 â€” Communication)
