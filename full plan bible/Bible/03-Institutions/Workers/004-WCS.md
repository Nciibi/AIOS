| Pattern | Description | Use Case | Authorization |
|---------|-------------|----------|---------------|
| **Direct** | Worker A sends to Worker B by session_id | Task handoff, result notification | Both in same Mission or Council-authorized |


## Message Delivery Guarantees

| Guarantee | Description | Retry Policy | Use Case |
|-----------|-------------|--------------|----------|
| **At Most Once** | Message delivered zero or one times. No retry on failure. | None | Log messages, non-critical updates |
| **At Least Once** | Message delivered one or more times. Retry on failure. | 3 retries with exponential backoff | Task results, status updates |
| **Exactly Once** | Message delivered exactly once. Deduplication. | 3 retries + idempotency check | Financial operations, critical coordination |

## Queuing

When a target Worker is busy or unavailable, WCS queues messages:

| Queue Property | Value |
|---------------|-------|
| Queue type | Per-Worker FIFO with priority sub-queues |
| Max queue depth | 1000 messages per Worker |
| Message TTL | Configurable (default: 1 hour) |
| Overflow policy | Oldest message dropped (with warning Event) |
| Priority ordering | critical â†’ high â†’ normal â†’ low |
| Delivery on reconnect | Queued messages delivered FIFO within priority |

## Cross-Cutting Concerns

### Security

All inter-Worker communication is authenticated and authorized. ACF message source identity is verified by WSS. Cross-Mission communication requires Security Council authorization. All communication attempts â€” including denied ones â€” produce Events for audit. (Physics/008-Security.md)

### Evidence

Every WCS operation â€” send, deliver, failure, subscription â€” produces an Event. Complete communication histories are preserved in the Event Store for audit and replay. (PHI-008)

### Lifecycle

WCS operates only for Workers in Running state. Workers in Paused, Blocked, or Failed states cannot send messages. Incoming messages to non-Running Workers are queued (for Paused) or rejected (for Failed/Terminated). (PHI-006)

### Capability Bounds

Communication capabilities are part of the Worker's CapabilitySet (determined by Genome + Org + Mission). A Worker cannot send message types it does not have capability for. Cross-Mission communication requires explicit capability grant. (Physics/007-Capabilities.md)

### Communication

All WCS operations use ACF as the underlying transport. WCS adds Worker-level routing, queuing, and authorization on top of ACF. Workers never communicate outside ACF. (Physics/009-Interaction.md, Law 3)
