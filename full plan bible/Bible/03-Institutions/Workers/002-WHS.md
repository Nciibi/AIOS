## Purpose

The Worker Health Service (WHS) monitors the operational health of Worker Sessions, handles timeouts, and manages graceful shutdown. WHS ensures that Workers are executing correctly, detects when Workers become unresponsive or unhealthy, and takes appropriate action â€” from pausing an idle Worker to force-terminating a failed one.

## Health Metrics

| Metric | Description | Measurement | Healthy Threshold |
|--------|-------------|-------------|-------------------|
| Heart Rate | Frequency of Worker actions (heartbeat signals) | Actions per minute | Configurable per template (default: â‰¥1/min) |
| Response Latency | Time for Worker to respond to ACF messages | Round-trip milliseconds | <5000ms |
| Error Rate | Proportion of actions resulting in errors | Errors / Total actions | <5% |
| Resource Usage | CPU, memory, tokens consumed vs budget | Current / Budget limit | <80% |
| Token Consumption Rate | Rate of authentication token use | Tokens / minute | Within budget burn rate |

## Health States

```
Healthy â†’ Degraded â†’ Unresponsive â†’ Failed
```

| State | Description | Detection Criteria | Actions |
|-------|-------------|-------------------|---------|
| **Healthy** | Worker is operating normally. All metrics within thresholds. | All metrics in range | Routine monitoring |
| **Degraded** | Worker is operational but showing signs of stress. | Any metric exceeds warning threshold (e.g., latency >3000ms, error rate >3%) | Log warning, notify Org, begin diagnostic |
| **Unresponsive** | Worker is not responding to heartbeats or ACF messages. | No heartbeat for N seconds (configurable, default: 30s) or no response to 3 consecutive probes | Escalate to Org Admin, attempt recovery |
| **Failed** | Worker has encountered an unrecoverable error. | Error rate >20%, or resources exhausted, or Unresponsive for >60s | Initiate graceful shutdown or force termination |

### State Transitions

| Transition | Trigger | Authorized By |
|-----------|---------|--------------|
| Healthy â†’ Degraded | Metrics exceed warning threshold | Automatic (WHS) |
| Degraded â†’ Healthy | Metrics return to normal for >60s | Automatic (WHS) |
| Degraded â†’ Unresponsive | Heartbeat timeout (no heartbeat for 30s) | Automatic (WHS) |
| Unresponsive â†’ Degraded | Worker resumes responding to probes | Automatic (WHS) |
| Unresponsive â†’ Failed | Recovery attempts exhausted (3 attempts) | Automatic (WHS) |
| Degraded â†’ Failed | Error rate >20% | Automatic (WHS) |

## Timeout Handling

| Timeout Type | Default Duration | Trigger | Action |
|-------------|-----------------|---------|--------|
| **Idle Timeout** | 15 minutes of no action | No heartbeat or action for threshold duration | Pause Worker (Running â†’ Paused), preserve context |
| **Total Runtime Cap** | Configurable per template (default: 24 hours) | Session exceeds max lifetime | Initiate graceful shutdown (Review â†’ Complete â†’ Archive) |
| **Response Timeout** | 30 seconds per ACF message | Worker does not respond to ACF message within threshold | Retry (3 attempts), then mark as Unresponsive |
| **Heartbeat Timeout** | 30 seconds | No heartbeat received within threshold | Mark as Unresponsive, begin recovery |

## Graceful Shutdown Flow

```
1. Shutdown trigger detected (runtime cap reached, Org command, Mission complete)
2. WHS sends SIGTERM-equivalent signal to Worker via ACF
3. Worker receives signal:
   a. Pause current task execution
   b. Save context (workspace, state, intermediate results)
   c. Seal evidence (produce final Events)
   d. Release resources (return to ROS)
   e. Acknowledge shutdown readiness
4. WHS confirms context saved and evidence sealed
5. WHS transitions Worker lifecycle state:
   Running â†’ Review (if Mission task complete)
   Running â†’ Completed (if no review needed)
   Running â†’ Archived (if immediate archival)
6. WHS revokes authentication tokens (ATS)
7. WHS notifies Org, Mission, and Security Council
8. Worker record marked as terminated in Registry
```

### Force Termination Flow

```
1. Worker is Unresponsive or Failed (no response to SIGTERM within 30s)
2. WHS escalates to Org Admin or Security Council
3. Authorizing entity confirms force termination
4. WHS:
   a. Runtime force-terminates the Session
   b. Context is preserved (may be partial)
   c. Evidence sealed with "force_terminated" flag
   d. Resources force-released
   e. Tokens force-revoked
5. WHS transitions Worker lifecycle state to Failed â†’ Archived
6. Security Event recorded: WHS.ForceTerminated
```

## Recovery Attempts

When a Worker becomes Unresponsive, WHS attempts recovery:

| Attempt | Action | Wait Time |
|---------|--------|-----------|
| 1 | Re-send last ACF message (retry) | 5 seconds |
| 2 | Send diagnostic probe | 10 seconds |
| 3 | Send restart signal (Runtime re-initialization) | 15 seconds |
| 4 (final) | Escalate to Org Admin for decision | Immediate |

If all 3 recovery attempts fail, the Worker transitions to Failed state and force termination is initiated.

## Cross-Cutting Concerns

### Security

Force termination requires Org Admin or Security Council authorization. Health state transitions to Unresponsive/Failed are recorded as Security Events. Unauthorized shutdown attempts are escalated. (Physics/008-Security.md)

### Evidence

Every health operation â€” heartbeat, state change, timeout, shutdown, termination â€” produces an Event. Complete health history of every Worker is recorded in the Event Store. Health events are consumed by OPE for Organization performance evaluation. (PHI-008)

### Lifecycle

WHS health states interact with the Worker lifecycle. Unresponsive Workers trigger lifecycle transitions (Running â†’ Failed). Graceful shutdown triggers lifecycle completion (Running â†’ Review â†’ Completed â†’ Archived). Timeout handling transitions idle Workers to Paused. (PHI-006)

### Capability Bounds

WHS operations are bounded by Worker capability scope. WHS cannot force-terminate a Worker outside its Organization's authority. Recovery actions are limited to constitutional bounds â€” no recovery action may violate Worker isolation or security boundaries. (Physics/007-Capabilities.md)

### Communication

All WHS operations communicate through ACF. Heartbeat signals, health probes, and shutdown commands are ACF messages. Health state change notifications are published to ACF event streams for Org Admin, Security Council, and OPE consumption. (Law 3 â€” Communication)
