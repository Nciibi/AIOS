## Purpose

IXP enables cross-instance identity resolution and federation. Instances resolve identities and verify credentials across instance boundaries. IXP is the foundation of trust for all other federation protocols.

## Protocol Flow

```
Requestor Instance                  Identity Instance
   │                                   │
   │──── resolveIdentity(entity_id) ──►│
   │◄── authenticate_instance ─────────│
   │──── instance_assertion ──────────►│
   │◄── authorize_query ───────────────│
   │──── query_authorized ────────────►│
   │◄── return_identity(record) ───────│
   │                                    │
   │    (verify trust chain)            │
   │    ├─ CSP.verify(attestation)      │
   │    ├─ trust_chain_check()          │
   │    └─ cache_identity()             │
   │                                    │
   │──── verify_credential ────────────►│
   │◄── verified ───────────────────────│
```

## Operations

| Operation | Description | Parameters |
|-----------|-------------|------------|
| resolveIdentity | Resolve remote identity | entity_id: EntityID, instance_id: InstanceID |
| verifyIdentity | Verify identity authenticity | identity_record: IdentityRecord |
| cacheIdentity | Cache resolved identity | identity_record: IdentityRecord, ttl: Duration |
| invalidateCache | Remove cached identity | entity_id: EntityID |
| getTrustChain | Get trust chain for instance | instance_id: InstanceID |

## Identity Resolution Response

| Field | Type | Description |
|-------|------|-------------|
| entity_id | EntityID | Resolved entity identifier |
| identity_record | IdentityRecord | Full identity from source instance |
| instance_attestation | bytes | Source instance attestation (CSP signed) |
| trust_chain | TrustChain | Chain of trust: Root CA → Instance CA → Entity |
| trust_level | TrustLevel | Resolved trust level |
| resolved_at | Timestamp | Resolution timestamp |

## Trust Chain

The trust chain establishes the cryptographic path from the resolving instance's trust anchor to the remote identity:

```
Resolving Instance Root CA
    └── Remote Instance CA (verified via IXP exchange)
            └── Entity Identity (signed by Remote Instance CA)
```

Trust chain verification:
1. Resolving instance has Remote Instance CA certificate (from prior IXP exchange or PXP discovery)
2. Remote Instance CA certificate signed by Remote Instance Root CA (exchanged through federated CA trust)
3. Entity identity signed by Remote Instance CA
4. All signatures verified through CSP

## Identity Caching

Resolved identities are cached to reduce cross-instance lookups:

| Aspect | Specification |
|--------|--------------|
| Default TTL | 5 minutes |
| Maximum TTL | 1 hour |
| Cache Size | 10,000 entries |
| Eviction Policy | LRU |
| Cache invalidation | On entity lifecycle change (pushed by source instance) |

## IXP Events

| Event Type | Produced When | Fields |
|-----------|--------------|--------|
| IXP.IdentityResolved | Remote identity resolved | entity_id, instance_id, trust_level |
| IXP.IdentityVerified | Identity authenticity verified | entity_id, verification_result |
| IXP.IdentityCached | Identity added to cache | entity_id, ttl |
| IXP.CacheInvalidated | Cache entry removed | entity_id, reason |
| IXP.TrustChainUpdated | Trust chain modified | instance_id, ca_cert_fingerprint |
| IXP.IdentityUnresolvable | Resolution failed | entity_id, instance_id, reason |

## Cross-Cutting Concerns

### Security
Identity records are signed by the source instance. Trust chains are verified through CSP before acceptance. Cache poisoning is prevented through signature verification on each cache entry.

### Evidence
Every identity resolution is logged. Verification results (success/failure) are recorded. Failed resolutions include reason codes.

### Lifecycle
Identity follows the IDS lifecycle. Cached identities are invalidated when the source entity's lifecycle changes (pushed by source instance through SXP).

### Capability Bounds
Instances may only resolve identities within their trust level scope. A low-trust instance may only resolve basic identity attributes, not full credential details.

### Design DNA Compliance
| Rule | Compliance |
|------|------------|
| R1 | IXP handles identity resolution and federation only. |
| R5 | Remote identity records implement the same IdentityRecord interface as local. |
| R9 | Identity resolution is deterministic for identical inputs (same entity, same instance). |
| R10 | Linear flow: resolve → verify → cache. |
| R13 | Remote instance unavailable → cached identity used if within TTL; otherwise denied. |
