## Purpose

OXP enables Organizations on different AIOS instances to form partnerships. Partnerships define resource sharing limits, knowledge sharing scope, mission collaboration terms, and governance alignment.

## Protocol Flow

```
Instance A Org                    Instance B Org
   │                                   │
   │──── proposePartnership ──────────►│
   │◄── negotiate_terms ───────────────│
   │──── counter_offer ───────────────►│
   │◄── accept ────────────────────────│
   │                                    │
   │──── establish_bridge ─────────────►│
   │◄── bridge_established ────────────│
   │                                    │
   │──── operate ──────────────────────►│
   │◄── operational_status ────────────│
```

## Partnership Terms

| Field | Type | Description |
|-------|------|-------------|
| partnership_id | UUID v4 | Unique partnership identifier |
| org_a_id | OrganizationID | Organization on Instance A |
| org_b_id | OrganizationID | Organization on Instance B |
| resource_sharing_limit | ResourceLimit[] | Max resources each may share |
| knowledge_sharing_scope | KnowledgeType[] | Types of knowledge shareable |
| mission_collaboration | boolean | Whether missions may span instances |
| governance_alignment | GovernanceTerms | Constitutional alignment requirements |
| duration | Duration | Partnership validity period |
| termination_policy | TerminationClause | How partnership may be ended |

## Operations

| Operation | Description | Parameters |
|-----------|-------------|------------|
| proposePartnership | Initiate partnership proposal | org_id: OrganizationID, target_instance: InstanceID, terms: PartnershipTerms |
| acceptPartnership | Accept partnership proposal | proposal_id: ProposalID |
| terminatePartnership | End partnership agreement | partnership_id: PartnershipID, reason: string |
| listPartnerships | List active partnerships | org_id: OrganizationID |
| getPartnershipStatus | Check partnership state | partnership_id: PartnershipID |

## Partnership Lifecycle

```
Proposed → Negotiating → Accepted → Active → Terminated
              │                         │
              └── Rejected ─────────────┘
```

| State | Description |
|-------|-------------|
| Proposed | Organization A sends proposal to Organization B |
| Negotiating | Terms being discussed (may include multiple rounds) |
| Accepted | Both organizations agree to terms |
| Active | Partnership is operational; resource/knowledge sharing enabled |
| Terminated | Partnership ended by either party |

## Operational Bridge

Once a partnership is Active, an operational bridge is established through CXP. The bridge enables:

- Resource sharing (RXP) between the organizations' instances
- Knowledge sharing (KXP) scoped to the agreed types
- Mission collaboration (MXP) for joint missions
- Security event sharing (SXP) between Security Councils

## OXP Events

| Event Type | Produced When | Fields |
|-----------|--------------|--------|
| OXP.PartnershipProposed | Proposal sent | proposal_id, org_ids, terms |
| OXP.PartnershipAccepted | Proposal accepted | partnership_id, accepted_terms |
| OXP.PartnershipTerminated | Partnership ended | partnership_id, reason, terminated_by |
| OXP.PartnershipExpired | Partnership duration elapsed | partnership_id |
| OXP.TermsModified | Partnership terms updated | partnership_id, modified_fields |

## Cross-Cutting Concerns

### Security
Partnership proposals are signed by the proposing organization's identity. Acceptance requires authorized signatories. Termination is irreversible.

### Evidence
Each partnership state transition produces events in both instances. Terms are recorded immutably.

### Lifecycle
Partnerships have defined duration. Expired partnerships require renewal. Termination triggers cleanup of all shared resources.

### Capability Bounds
Organizations may only enter partnerships within their constitutional authority. An Organization cannot agree to resource sharing that exceeds its constitutional limits.

### Design DNA Compliance
| Rule | Compliance |
|------|------------|
| R1 | OXP handles Organization partnerships only. |
| R10 | Linear flow: propose → negotiate → accept → operate → terminate. |
