## Purpose

GXP enables cross-instance Genome (Template) sharing. Templates define entity types, capabilities, and behaviours. GXP ensures that shared Genomes are verified, constitutionally consistent, and privacy-compliant before import.

## Protocol Flow

```
Publisher                           Subscriber
   │                                   │
   │──── publishGenome(genome_id) ─────►│
   │                                    │
   │    (subscriber verifies)           │
   │    ├─ CSP.verify(signature)        │
   │    ├─ constitutional_check()       │
   │    └─ privacy_compliance()         │
   │                                    │
   │◄── requestGenome(genome_id) ───────│
   │──── transfer(genome_package) ─────►│
   │◄── acknowledge(import_result) ─────│
```

## Operations

| Operation | Description | Parameters |
|-----------|-------------|------------|
| publishGenome | Announce Genome availability | genome_id: GenomeID, instance_id: InstanceID |
| requestGenome | Request Genome from publisher | genome_id: GenomeID, publisher_id: InstanceID |
| importGenome | Import shared Genome locally | genome_id: GenomeID, genome_package: GenomePackage |
| listSharedGenomes | List available shared Genomes | filter: GenomeFilter |
| deprecateSharedGenome | Mark shared Genome as deprecated | genome_id: GenomeID, reason: string |

## Genome Package

| Field | Type | Description |
|-------|------|-------------|
| package_id | UUID v4 | Unique package identifier |
| genome_id | GenomeID | Source Genome identifier |
| genome_definition | bytes | Serialized Genome (AGS format) |
| version | string | Semantic version |
| publisher_id | InstanceID | Publishing instance |
| signature | bytes | AGS signing key signature (Ed25519) |
| constitutional_hash | SHA-256 | Hash of constitutional consistency check |
| privacy_assertion | bytes | Privacy compliance attestation |
| dependencies | GenomeID[] | Required dependent Genomes |

## Import Validation

Before a Genome is imported, the subscriber instance performs:

1. **Signature Verification**: CSP.verify(genome_package.signature, genome_definition, publisher_public_key)
2. **Constitutional Consistency Check**: The Genome does not violate any constitutional law of the importing instance
3. **Privacy Compliance**: The Genome does not request capabilities that violate CPR-010
4. **Dependency Resolution**: All dependent Genomes are available (locally or through GXP)

## Genome Sharing Rules

- All Genomes are signed by the AGS of the publishing instance
- Constitutional consistency check runs against the importing instance's constitution (may differ from publisher)
- Genomes imported from federation are marked as `federated` in the local AGS registry
- Federated Genomes may be depublished by the publisher at any time
- Deprecated Genomes remain in local registries but are flagged for replacement

## GXP Error Codes

| Code | Condition | Recovery |
|------|-----------|----------|
| GXP_001 | Genome signature verification failed | Request re-signed genome from publisher |
| GXP_002 | Constitutional consistency check failed | Genome incompatible with local constitution |
| GXP_003 | Privacy compliance check failed | Genome requests capabilities violating CPR-010 |
| GXP_004 | Dependency genome not found | Resolve dependencies via GXP first |
| GXP_005 | Genome deprecated by publisher | Find alternative genome |
| GXP_006 | Genome version conflict | Negotiate compatible version |

## Cross-Cutting Concerns

### Security
Genomes are signed by the publisher's AGS. Import verification includes signature, constitutional, and privacy checks. Tampered genomes are rejected.

### Evidence
All Genome exchanges are logged. Import results include verification evidence. Deprecation events are stored.

### Lifecycle
Genomes follow AGS lifecycle. Federated Genomes may be independently versioned by each instance.

### Capability Bounds
Instances may only import Genomes compatible with their constitutional framework. A Genome requiring capabilities the instance does not support is rejected.
