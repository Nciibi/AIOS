## Purpose

KXP enables cross-instance knowledge sharing through the Academy. Knowledge is shared subject to privacy policy (CPR-010), origin instance trust level, and knowledge type classification.

## Knowledge Classification

| Type | Description | Sharable? | Consent Required |
|------|-------------|-----------|-----------------|
| Constitutional | Laws, Physics, Bible documents | Always | No |
| Operational | Missions, processes, workflows | By agreement | Yes |
| Performance | Entity metrics, resource usage | By agreement | Yes |
| Research | Academy research output | With consent | Yes |
| Private | Entity-specific data | Never | N/A |

## Protocol Flow

```
Requestor                          Provider
   │                                   │
   │──── requestKnowledge(type,id) ───►│
   │◄── authenticate_request ──────────│
   │──── identity_assertion ──────────►│
   │◄── authorize_scope ───────────────│
   │──── accept_scope ────────────────►│
   │◄── transfer(knowledge_package) ───│
   │──── verify(knowledge_hash) ──────►│
   │◄── acknowledge ───────────────────│
```

## Operations

| Operation | Description | Parameters |
|-----------|-------------|------------|
| shareKnowledge | Push knowledge to peer instance | knowledge_id: KnowledgeID, target_instance: InstanceID |
| requestKnowledge | Request knowledge from peer | knowledge_type: KnowledgeType, knowledge_id: KnowledgeID |
| subscribeKnowledge | Subscribe to knowledge updates | knowledge_type: KnowledgeType, filter: KnowledgeFilter |
| unsubscribeKnowledge | Cancel subscription | subscription_id: SubscriptionID |
| getSharedKnowledge | List shared knowledge catalog | instance_id: InstanceID |

## Knowledge Package

| Field | Type | Description |
|-------|------|-------------|
| package_id | UUID v4 | Unique package identifier |
| knowledge_id | KnowledgeID | Source knowledge identifier |
| knowledge_type | KnowledgeType | Classification type |
| payload | bytes | Serialized knowledge content |
| origin_instance | InstanceID | Source instance |
| signature | bytes | Origin instance CSP signature |
| privacy_attestation | bytes | Signed privacy compliance assertion |
| timestamp | Timestamp | ISO 8601 |

## Knowledge Sharing Rules

1. Constitutional knowledge is always shareable without restriction
2. Operational knowledge requires an active OXP partnership agreement
3. Research knowledge requires entity consent (opt-in)
4. Private knowledge is never shareable
5. All knowledge is subject to the destination instance's trust level
6. Knowledge packages must include a valid privacy attestation

## Cross-Cutting Concerns

### Security
Knowledge packages are signed by the origin instance. Recipients verify signatures before acceptance. Privacy attestation ensures CPR-010 compliance.

### Evidence
Every knowledge transfer produces events in both instances. Knowledge access is audited.

### Lifecycle
Knowledge has a lifecycle through the Academy. Shared knowledge is versioned. Deprecated knowledge is removed from the share catalog.

### Capability Bounds
Instances may only request knowledge types their trust level permits. A low-trust instance may only access constitutional knowledge.
