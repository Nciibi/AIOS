## Purpose

TXP enables cross-instance resource trading and automated accounting. Instances may trade resources (tokens, compute credits, storage quotas) using AIOS Credits (AIC) as the exchange unit. Trading is fully automated with settlement and audit trails.

## Trading Unit

AIOS Credit (AIC) is the standard trading unit:

| Resource | Conversion | 1 AIC Equals |
|----------|-----------|--------------|
| Token Capacity | 1000 tokens | 1,000 tokens |
| Compute | 1 GHz-hour | 3,600,000 GHz-ms |
| Storage | 10 GB-days | 864,000 GB-seconds |

Exchange rates are configurable per partnership (OXP) with default rates as above.

## Protocol Flow

```
Seller Instance                     Buyer Instance
   │                                   │
   │──── postOffer(resource,price) ───►│
   │                                    │
   │◄── discoverOffers(filter) ────────│
   │──── offer_list ──────────────────►│
   │                                    │
   │◄── negotiate(offer_id,counter) ───│
   │──── accept(counter) ─────────────►│
   │                                    │
   │──── executeTrade(trade_id) ───────►│
   │◄── trade_executed ─────────────────│
   │                                    │
   │──── settle(trade_id) ─────────────►│
   │◄── settled(transaction_ref) ───────│
```

## Operations

| Operation | Description | Parameters |
|-----------|-------------|------------|
| postOffer | Post resource for sale | resource: ResourceType, quantity: uint, price_aic: uint, terms: TradeTerms |
| discoverOffers | Search available offers | filter: OfferFilter |
| negotiateTrade | Counter-offer or accept price | offer_id: OfferID, counter_price: uint |
| executeTrade | Execute agreed trade | trade_id: TradeID |
| settleTrade | Settle trade balance | trade_id: TradeID |
| getBalance | Check AIC balance | instance_id: InstanceID |
| getTransactionHistory | List past transactions | instance_id: InstanceID, limit: uint |

## Offer Format

| Field | Type | Description |
|-------|------|-------------|
| offer_id | UUID v4 | Unique offer identifier |
| seller_id | InstanceID | Selling instance |
| resource_type | ResourceType | Type of resource |
| quantity | uint | Amount available |
| unit_price_aic | uint | Price per unit in AIC |
| minimum_order | uint | Minimum trade quantity |
| valid_until | Timestamp | Offer expiry |
| signature | bytes | Seller CSP signature |

## Trade Lifecycle

```
Offer → Negotiation → Agreement → Execution → Settlement → Complete
                                                      │
                                                      └── Dispute
```

| State | Description |
|-------|-------------|
| Offer | Resource listed for sale |
| Negotiation | Price negotiation in progress |
| Agreement | Both parties agree to terms |
| Execution | Resource transferred |
| Settlement | AIC balance adjusted |
| Complete | Trade fully settled |
| Dispute | One party contests settlement |

## TXP Error Codes

| Code | Condition | Recovery |
|------|-----------|----------|
| TXP_001 | Insufficient AIC balance | Acquire credits before trading |
| TXP_002 | Offer expired | Request new offer from seller |
| TXP_003 | Trade execution failed | Resource unavailable or capacity changed |
| TXP_004 | Settlement failed | Verify AIC ledger integrity |
| TXP_005 | Dispute resolution timeout | Escalate to both Security Councils |
| TXP_006 | Exchange rate not agreed | Negotiate rate before execution |

## TXP Events

| Event Type | Produced When | Fields |
|-----------|--------------|--------|
| TXP.OfferPosted | New resource offer | offer_id, resource_type, quantity, price |
| TXP.OfferAccepted | Offer accepted | offer_id, buyer_id, agreed_price |
| TXP.TradeExecuted | Resource transferred | trade_id, resource_type, quantity |
| TXP.TradeSettled | Credits transferred | trade_id, amount_aic, from_instance, to_instance |
| TXP.DisputeRaised | Trade dispute | trade_id, reason, disputing_party |

## Cross-Cutting Concerns

### Security
Offers and trades are signed by both parties. Settlement requires dual signatures. Disputes are escalated to both instances' Security Councils.

### Evidence
Every trade state transition is logged. Settlement records are immutably stored. Transaction history is available for audit.

### Lifecycle
Trades have a defined lifecycle with expiration. Unsettled trades after 24 hours trigger automatic dispute.

### Capability Bounds
Instances may only trade resources within their constitutional limits. An instance cannot sell resources it does not own.
