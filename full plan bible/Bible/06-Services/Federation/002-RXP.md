## Purpose

RXP enables cross-instance resource sharing. Instances may borrow tokens, compute capacity, and storage from trusted peer instances. Resource usage is tracked, accounted, and billed between instances.

## Protocol Flow

```
Requestor                          Provider
   │                                   │
   │──── requestResource(type,qty) ──►│
   │◄── offer(capacity,price) ────────│
   │──── accept(offer_id) ───────────►│
   │◄── transfer(resource_token) ─────│
   │──── use ────────────────────────►│
   │──── return(resource_token) ─────►│
   │◄── account(usage,balance) ───────│
```

## Resource Limits

| Resource | Max Shareable | Unit |
|----------|---------------|------|
| Token Capacity | 20% of provider capacity | Tokens/second |
| Compute | 20% of available cycles | GHz-hours |
| Storage | 20% of available capacity | GB-days |
| Memory | 10% of available RAM | GB-hours |

The 20% limit is configurable per partnership agreement (OXP). Default is 20%.

## Operations

| Operation | Description | Parameters |
|-----------|-------------|------------|
| requestResource | Request resource availability | type: ResourceType, quantity: uint, duration: Duration |
| offerResource | Offer available resources | request_id: RequestID, capacity: Capacity, terms: Terms |
| acceptOffer | Accept a resource offer | offer_id: OfferID |
| transferResource | Transfer resource to requestor | offer_id: OfferID, resource_token: ResourceToken |
| returnResource | Return borrowed resource | resource_token: ResourceToken |
| getResourceBalance | Check resource account balance | instance_id: InstanceID |

## Resource Token

| Field | Type | Description |
|-------|------|-------------|
| token_id | UUID v4 | Unique token identifier |
| resource_type | ResourceType | type of resource allocated |
| quantity | uint | Amount allocated |
| allocated_at | Timestamp | When resource was allocated |
| expires_at | Timestamp | When allocation expires |
| provider_id | InstanceID | Provider instance |
| consumer_id | InstanceID | Consumer instance |
| signature | bytes | Provider CSP signature |

## Accounting

Resource usage is tracked per instance pair. Accounting records:

| Field | Description |
|-------|-------------|
| period_start | ISO 8601 start of billing period |
| period_end | ISO 8601 end of billing period |
| resources_consumed | Quantity of each resource type used |
| balance_due | Net credits owed (positive = consumer owes) |
| settled | Whether balance has been paid |

## RXP Events

| Event Type | Produced When | Fields |
|-----------|--------------|--------|
| RXP.ResourceRequested | Resource request sent | request_id, resource_type, quantity |
| RXP.ResourceOffered | Provider responds with offer | offer_id, capacity, terms |
| RXP.ResourceTransferred | Resource allocated to requestor | token_id, resource_type, quantity |
| RXP.ResourceReturned | Resource returned to provider | token_id, usage_record |
| RXP.ResourceExhausted | Provider reaches capacity limit | resource_type, max_shareable, current_share |

## Cross-Cutting Concerns

### Security
Resource tokens are signed by the provider instance's CSP. Token reuse is prevented through unique token_id and expiry. Exhausted resources trigger alerts to both instances' ROS.

### Evidence
Every resource transaction produces events in both instances. Accounting records are immutably stored.

### Lifecycle
Resource allocations have a fixed lifespan (expires_at). Automatic return on expiry. Overdue resources trigger escalation.

### Capability Bounds
Provider instances cannot exceed 20% capacity sharing limit. Consumer instances are bounded by their ROS capability profile.

### Design DNA Compliance
| Rule | Compliance |
|------|------------|
| R1 | RXP handles resource exchange only. |
| R10 | Simple request → offer → accept → use → return flow. |
| R13 | Provider exhausted → request denied with capacity info. |
