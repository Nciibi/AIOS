## Purpose

CXP establishes and manages the ACF inter-instance bridge. CXP is the transport layer for all other federation protocols. It manages handshake, authentication, bridge lifecycle, encryption configuration, heartbeat monitoring, and automatic reconnection.

## Bridge Lifecycle

```
Handshake → Authenticate → Established → Monitoring → Teardown
                │                              │
                └── Failed ────────────────────┘
```

| State | Description |
|-------|-------------|
| Handshake | Protocol version negotiation, capability exchange |
| Authenticate | Mutual authentication via mTLS + IXP |
| Established | Bridge operational, traffic flowing |
| Monitoring | Active health checks, latency measurement |
| Teardown | Graceful shutdown, resource cleanup |
| Failed | Bridge connection lost, attempting reconnect |

## Protocol Flow

```
Instance A                          Instance B
   │                                   │
   │──── handshake(version, caps) ────►│
   │◄── handshake_ack(version, caps) ──│
   │                                    │
   │──── authenticate(identity_token) ►│
   │◄── authenticate_ack(identity) ─────│
   │                                    │
   │──── establish_bridge(config) ─────►│
   │◄── bridge_established(ack) ────────│
   │                                    │
   │──── heartbeat(seq) ───────────────►│
   │◄── heartbeat_ack(seq) ─────────────│
   │                                    │
   │──── teardown(reason) ─────────────►│
   │◄── teardown_ack ───────────────────│
```

## Bridge Configuration

| Field | Type | Description | Default |
|-------|------|-------------|---------|
| bandwidth_limit | uint | Max Mbps | 1000 |
| latency_target | uint | Max RTT in ms | 100 |
| encryption | EncryptionMode | mTLS + CSP | required |
| heartbeat_interval | uint | Seconds between heartbeats | 5 |
| heartbeat_timeout | uint | Seconds before declaring failure | 15 |
| auto_reconnect | boolean | Automatic reconnection | true |
| max_retries | uint | Max reconnect attempts | 5 |

## Operations

| Operation | Description | Parameters |
|-----------|-------------|------------|
| establishBridge | Create inter-instance bridge | instance_id: InstanceID, config: BridgeConfig |
| configureBridge | Update bridge configuration | bridge_id: BridgeID, config: BridgeConfig |
| getBridgeStatus | Query bridge health | bridge_id: BridgeID |
| reconnectBridge | Trigger reconnection | bridge_id: BridgeID |
| teardownBridge | Destroy inter-instance bridge | bridge_id: BridgeID, reason: string |

## Heartbeat Protocol

Heartbeats are sent at the configured interval (default 5 seconds):

1. Instance A sends heartbeat(seq_num, timestamp)
2. Instance B responds heartbeat_ack(seq_num, timestamp)
3. Instance A measures RTT = response_time - sent_time
4. If RTT > latency_target: log warning, adjust QoS
5. If no response within heartbeat_timeout: declare bridge failed
6. On failure: attempt reconnect up to max_retries times
7. After max_retries: escalate to Security Council, preserve bridge metadata

## CXP Error Codes

| Code | Condition | Recovery |
|------|-----------|----------|
| CXP_001 | Handshake version mismatch | Negotiate common protocol version |
| CXP_002 | Authentication failed during bridge setup | Verify identity tokens with IXP |
| CXP_003 | Bridge configuration invalid | Correct configuration parameters |
| CXP_004 | Heartbeat timeout | Check network connectivity |
| CXP_005 | Max retries exceeded | Escalate to Security Council |
| CXP_006 | Bridge not found | Verify bridge_id |

## Cross-Cutting Concerns

### Security
All bridge traffic uses mTLS with mutual certificate verification. Encryption is mandatory. Heartbeat includes CSP-signed nonces to prevent replay attacks.

### Evidence
Every bridge state transition produces events. Heartbeat latency is logged for performance analysis. Bridge failure events are stored for post-mortem.

### Lifecycle
Bridges follow the defined lifecycle. Tear down is graceful with resource cleanup. Bridge metadata is retained for 7 days after teardown.

### Capability Bounds
Bridge bandwidth is bounded by configuration and instance capacity. An instance cannot establish more bridges than its ACF capacity allows.
