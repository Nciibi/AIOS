## Purpose

SXP enables Security Councils on different AIOS instances to share threat intelligence and coordinate security responses. Threat data is shared in real-time for active threats and periodically for intelligence reports.

## Threat Types

| Threat Type | Description | Severity | Example |
|-------------|-------------|----------|---------|
| Identity Compromise | An entity identity is compromised | Critical | Stolen signing key |
| Token Theft | Authentication tokens leaked | High | Session token exfiltration |
| Policy Violation | Constitutional or policy breach | Medium | Unauthorized resource access |
| Attack Pattern | Observed attack methodology | Medium | Novel attack vector |
| Vulnerability | Software or configuration weakness | Medium | Unpatched dependency |
| Anomaly | Suspicious but unconfirmed activity | Low | Unusual connection pattern |

## Protocol Flow

```
Reporting Instance                 Receiving Instance
   │                                   │
   │──── shareThreat(threat_data) ────►│
   │                                    │
   │    (receiving instance)            │
   │    ├─ verify(signature)            │
   │    ├─ assess(severity, scope)      │
   │    └─ respond or escalate          │
   │                                    │
   │◄── acknowledge(threat_id) ─────────│
   │◄── (optional) escalate_request ────│
   │──── escalate_response ────────────►│
```

## Operations

| Operation | Description | Parameters |
|-----------|-------------|------------|
| shareThreat | Share threat intelligence | threat_data: ThreatData, severity: Severity |
| subscribeThreats | Subscribe to threat feed | filter: ThreatFilter, callback_endpoint: ACFAddress |
| acknowledgeThreat | Acknowledge receipt of threat | threat_id: ThreatID, action_taken: string |
| escalateThreat | Request coordinated response | threat_id: ThreatID, escalation_reason: string |
| getThreatStatus | Query threat resolution status | threat_id: ThreatID |

## Threat Data Format

| Field | Type | Description |
|-------|------|-------------|
| threat_id | UUID v4 | Unique threat identifier |
| threat_type | ThreatType | Classification |
| severity | Severity | critical, high, medium, low |
| description | string | Human-readable threat description |
| indicators | Indicator[] | Observable indicators (IPs, hashes, patterns) |
| affected_entities | EntityID[] | Entities known to be affected |
| reporting_instance | InstanceID | Source instance |
| signature | bytes | Reporting instance CSP signature |
| timestamp | Timestamp | ISO 8601 |
| tlp | TLPLevel | Traffic Light Protocol level |

## Threat Response Coordination

| Severity | Response Time | Action |
|----------|--------------|--------|
| Critical | Immediate (<60s) | Automated block rules deployed, all instances notified |
| High | <5 minutes | Security Council review, targeted countermeasures |
| Medium | <1 hour | Investigation, intelligence enrichment |
| Low | <24 hours | Logged, periodic review |

## SXP Events

| Event Type | Produced When | Fields |
|-----------|--------------|--------|
| SXP.ThreatShared | Threat intelligence shared | threat_id, threat_type, severity |
| SXP.ThreatAcknowledged | Receipt acknowledged | threat_id, acknowledging_instance |
| SXP.ThreatEscalated | Threat escalated for coordination | threat_id, escalation_reason |
| SXP.ThreatResolved | Threat resolved | threat_id, resolution, actions_taken |
| SXP.SubscriptionCreated | Threat feed subscription | subscription_id, filter |

## Cross-Cutting Concerns

### Security
Threat data is signed by the reporting instance. TLP levels control distribution. Critical threats trigger automated responses.

### Evidence
All threat sharing is logged. Threat resolution evidence is shared on request.

### Lifecycle
Threats have a lifecycle: reported → acknowledged → investigated → resolved. Resolved threats remain in the intelligence database.

### Capability Bounds
Instances may only share threat types within their security capability scope. A low-trust instance may only receive threats, not share them.

### Design DNA Compliance
| Rule | Compliance |
|------|------------|
| R1 | SXP handles security event sharing only. |
| R10 | Linear flow: share → verify → assess → respond. |
| R13 | Threat feed unavailable → local security continues with local intelligence only. |
