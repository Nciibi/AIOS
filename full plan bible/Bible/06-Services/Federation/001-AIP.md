## Purpose

AIP enables cross-instance Session-to-Session communication. Workers and Sessions on Instance A can interact with Workers and Sessions on Instance B through a secure, authenticated protocol built on the ACF inter-instance bridge.

## Operations

| Operation | Description | Parameters |
|-----------|-------------|------------|
| connectToRemoteAgent | Initiate cross-instance session | remote_session_id: SessionID, instance_id: InstanceID |

## AIP Session Lifecycle

```
Connecting вҶ’ Authenticating вҶ’ Active вҶ’ Disconnecting вҶ’ Disconnected
     в”Ӯ                            в”Ӯ          в”Ӯ
     в””в”Җв”Җ Failed в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”ҳв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”ҳ
```

| State | Description | Owner |
|-------|-------------|-------|
| Connecting | Initial connection request sent | Initiator |
| Authenticating | Mutual identity verification | Both |
| Active | Messages can be exchanged | Both |
| Disconnecting | Graceful teardown in progress | Either |
| Disconnected | Session fully terminated | Neither |
| Failed | Connection lost unexpectedly | Both |

| Timeout | Duration | Action |

## Security Model

| Aspect | Specification |
|--------|--------------|
| Mutual Authentication | Both sessions present valid identity tokens signed by their respective instances |
| Capability Scoping | Remote session capabilities are scoped to the intersection of both sessions' profiles |
| Rate Limiting | Max 100 messages/second per session pair (configurable) |
| Audit Logging | All messages logged with correlation ID in both instances |
| Session Isolation | Each cross-instance session is isolated; no session can access another's data |

## Cross-Cutting Concerns

### Security
All AIP messages are signed by the sending session's identity key. Messages are encrypted through the ACF bridge (mTLS + CSP). Session isolation prevents cross-session data access.

### Evidence
Every AIP operation produces events in both instances. Messages are logged with correlation IDs for audit trail reconstruction.

### Lifecycle
AIP sessions follow Session lifecycle. Cross-instance sessions are terminated if either instance's bridge fails.

### Capability Bounds
Entities may only connect to remote sessions within their capability scope. A L0 entity may only receive messages, not send.
