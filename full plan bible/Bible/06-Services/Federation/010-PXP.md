## Purpose

PXP enables cross-instance platform capability advertisement and discovery. Instances advertise their available services, versions, capacities, and trust levels. Other instances discover and consume these capabilities.

## Capability Advertisement

| Field | Type | Description |
|-------|------|-------------|
| instance_id | InstanceID | Advertising instance |
| services[] | ServiceID[] | Available services |
| versions[] | VersionMap | Service version information |
| capacities[] | CapacityMap | Resource capacity limits |
| trust_level | TrustLevel | Instance trust rating |
| location | string | Geographic or network location |
| supported_protocols | ProtocolID[] | Federation protocols supported |
| advertised_at | Timestamp | Last advertisement update |

## Protocol Flow

```
Advertiser                         Discoverer
   │                                   │
   │──── advertiseCapabilities ────────►│
   │◄── discoverInstances(filter) ─────│
   │──── instance_list(response) ─────►│
   │                                    │
   │◄── getInstanceInfo(instance_id) ───│
   │──── instance_profile ─────────────►│
   │                                    │
   │◄── connectToService(service_id) ───│
   │──── service_endpoint ─────────────►│
   │◄── disconnectService(service_id) ──│
   │──── disconnected ─────────────────►│
```

## Operations

| Operation | Description | Parameters |
|-----------|-------------|------------|
| advertiseCapabilities | Broadcast capabilities | capabilities: CapabilityAdvertisement |
| discoverInstances | Find peer instances | filter: DiscoveryFilter |
| getInstanceInfo | Get detailed instance profile | instance_id: InstanceID |
| connectToService | Establish service connection | instance_id: InstanceID, service_id: ServiceID |
| disconnectService | Tear down service connection | instance_id: InstanceID, service_id: ServiceID |

## Discovery Filter

| Field | Type | Description |
|-------|------|-------------|
| service_filter | ServiceID[] | Only instances with these services |
| min_trust_level | TrustLevel | Minimum trust level |
| location_filter | string | Geographic region |
| capacity_min | CapacityMap | Minimum capacity requirements |
| protocol_filter | ProtocolID[] | Required federation protocols |

## Instance Trust Levels

| Level | Name | Description |
|-------|------|-------------|
| T0 | Unknown | No prior interaction |
| T1 | Verified | Identity verified, no prior federation |
| T2 | Partner | Active partnership (OXP) in place |
| T3 | Trusted | Long-term partnership, no incidents |
| T4 | Allied | Full mutual trust, joint governance |

## PXP Error Codes

| Code | Condition | Recovery |
|------|-----------|----------|
| PXP_001 | Instance not discoverable (filter mismatch) | Broaden discovery filter |
| PXP_002 | Advertisement signature invalid | Request re-signed advertisement |
| PXP_003 | Service not available on target instance | Check advertised services |
| PXP_004 | Trust level insufficient for service | Increase trust level through OXP |
| PXP_005 | Capability advertisement expired (24h) | Request fresh advertisement |
| PXP_006 | Service connection rejected by provider | Verify provider capacity |

## PXP Events

| Event Type | Produced When | Fields |
|-----------|--------------|--------|
| PXP.CapabilitiesAdvertised | Instance broadcasts capabilities | instance_id, services[], capacities[] |
| PXP.InstanceDiscovered | Peer instance found | instance_id, trust_level, services[] |
| PXP.ServiceConnected | Service connection established | instance_id, service_id, endpoint |
| PXP.ServiceDisconnected | Service connection torn down | instance_id, service_id, reason |
| PXP.TrustLevelChanged | Instance trust rating updated | instance_id, old_level, new_level |

## Cross-Cutting Concerns

### Security
Capability advertisements are signed by the advertising instance. Discovery results are verified before connection. Trust levels gate service access.

### Evidence
All discovery and connection operations are logged. Trust level changes are recorded with justification.

### Lifecycle
Advertisements expire after 24 hours. Instances must re-advertise to remain discoverable. Service connections persist until explicitly disconnected.

### Capability Bounds
Instances cannot advertise capabilities they do not possess. Advertised capacities must match actual capacity limits.

### Design DNA Compliance
| Rule | Compliance |
|------|------------|
| R1 | PXP handles capability discovery only. |
| R3 | Capability definitions from Platform Architecture (single source). |
| R10 | Simple advertise → discover → connect flow. |
| R12 | Every error has a unique code (PXP_001-006). |
| R13 | Instance unavailable → not discoverable; clients use cached info with staleness check. |
| R14 | Paved path: advertise → discover → connect → consume. No alternative. |
