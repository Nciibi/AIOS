## Purpose

EXP enables cross-instance evidence sharing for audit and constitutional processes. Instances may share Events for cross-instance audits, investigations, and constitutional review. Evidence sharing is gated by privacy policy, audit requirements, and Security Council authorization.

## Operations

| Operation | Description | Parameters |
|-----------|-------------|------------|
| requestEvidence | Request evidence from peer | scope: EvidenceScope, reason: string, authorization: Authorization |
| shareEvidence | Push evidence to peer | evidence: EvidencePackage, target_instance: InstanceID |
| verifyEvidence | Verify evidence chain integrity | evidence: EvidencePackage |
| subscribeEvidence | Subscribe to evidence stream | filter: EvidenceFilter |
| getEvidenceChain | Retrieve evidence chain | evidence_id: EvidenceID |

## Authorization Gates

Evidence sharing requires authorization at multiple levels:

| Gate | Check | Authority |
|------|-------|-----------|
| Identity | Requestor identity is valid | IDS |
| Purpose | Request reason is valid | Security Council |
| Scope | Requested evidence within policy scope | Policy System |
| Privacy | CPR-010 compliance check | Privacy Engine |
| Authorization | Security Council approval | Security Council |

## EXP Error Codes

| Code | Condition | Recovery |
|------|-----------|----------|
| EXP_001 | Evidence request not authorized | Obtain Security Council authorization |
| EXP_002 | Evidence scope exceeds privacy policy | Narrow request scope |
| EXP_003 | Evidence chain integrity check failed | Request re-verified evidence chain |
| EXP_004 | Privacy redaction not applied | Apply redactions per CPR-010 |
| EXP_005 | Evidence subscription filter invalid | Correct filter parameters |
| EXP_006 | Evidence not found on provider | Verify evidence_id scope |

## Cross-Cutting Concerns

### Security
Evidence is signed by the source instance. Chain integrity proof ensures events have not been tampered. Privacy redactions are recorded and verifiable.

### Evidence
Evidence exchange is itself evidenced. Every transfer produces events in both instances' Event Stores.

### Lifecycle
Evidence follows the Event Store lifecycle. Shared evidence may be verified independently by the receiving instance.

### Capability Bounds
Instances may only request evidence types within their authorization scope. Evidence requests without valid authorization are automatically denied.
