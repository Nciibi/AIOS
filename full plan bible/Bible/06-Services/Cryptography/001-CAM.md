## Purpose

CAM manages TLS certificates and internal AIOS certificate authorities. CAM is the sole entity that issues, renews, revokes, and distributes certificates. All TLS communication in AIOS uses certificates managed by CAM. CAM maintains an internal root CA for intra-instance communication and supports federated CA for cross-instance trust.

## Certificate Types

| Type | Purpose | Validity | Storage |
|------|---------|----------|---------|
| TLS Server | Internal service TLS termination | 90 days | HSM + Encrypted Store |
| TLS Client | Entity authentication for mTLS | 90 days | Encrypted Store |
| CA Root | Internal root of trust | 5 years | HSM (key never exported) |
| CA Intermediate | Subordinate CA for specific domains | 2 years | HSM |
| Code Signing | Genome signature verification | 1 year | HSM |
| Admin | Human operator authentication | 24 hours | Ephemeral |

### Certificate Request Flow

```
1. Entity generates key pair via CSP.generateKey(signing_algorithm)
2. Entity submits CSR to CAM.requestCertificate()
3. CAM validates: entity identity (via IDS), capability (via CCA), purpose
4. CAM checks: subject uniqueness, SAN validity, not_before constraints
5. CAM signs CSR via CSP.sign(csr_hash, ca_key_id)
6. CAM stores certificate in encrypted store
7. CAM publishes certificate_issued event
8. Entity receives certificate via ACF response
```

## Certificate Revocation List

CRL is published on ACF topic `acf/crypto/crl`. All services cache CRL with TTL of 1 hour. CAM issues a new CRL within 5 minutes of any revocation. CRL is signed by the issuing CA. Services that cannot reach the CRL topic within the TTL window must deny TLS connections (fail-closed per R13).

## Cross-Cutting Concerns

### Security
CA private keys never leave HSM. Root CA key is generated inside HSM and never exported. CAM issues certificates only to verified identities. Revocation is immediate on compromise notification.

### Evidence
Every certificate lifecycle transition produces a CAM event logged in the Event Store. Certificate issuance requires identity verification evidence.

### Lifecycle
Certificates follow a fixed lifecycle with automatic renewal at 30 days before expiry. CAM monitors all certificates and alerts Security Council on renewal failure.

### Capability Bounds
Entities may only request certificate types their capability profile allows. A Worker entity cannot request a CA certificate.

### Communication
All CAM operations use ACF with mTLS using existing valid certificates. CAM does not expose HTTP endpoints.
