## Purpose

Queryable audit trail over the Event Store. AUS provides filtered, searchable access to audit Events — all Events tagged with `audit: true`. Audit Events include security, identity, lifecycle, governance, and policy Events. AUS enables the Security Council, Organization managers, and entity supervisors to investigate system activity.

## Audit Events

Audit Events are all Events with the `audit: true` tag. The following Event categories are always audit Events:

| Category | Event Types | Retention |
|----------|-------------|-----------|
| **Security** | Authentication, Authorization, Verification, Escalation | 7 years |
| **Identity** | Created, Verified, Suspended, Restored, Retired, Archived | 7 years |
| **Lifecycle** | StateChanged, EntityCreated, EntityCompleted, TransitionDenied | 7 years |
| **Governance** | RFC Submitted, Constitutional Review, Policy Change, Amendment | 7 years |
| **Policy** | Policy Evaluated, Capability Granted, Capability Revoked | 5 years |

## AUS Operations

```
queryAuditLog(filter) → AuditRecord[]
queryAuditLogPaginated(filter, page, size) → PaginatedResult
getAuditTrail(entity_id) → AuditRecord[]
getAuditTrailByTime(entity_id, from, to) → AuditRecord[]
getSecurityEvents(time_range, severity?) → AuditRecord[]
exportAuditLog(format, filter) → export_handle
getExportStatus(export_handle) → ExportStatus
getAuditStats(time_range) → AuditStats
searchAuditLog(query) → AuditRecord[]
getAuditSummary(entity_id, time_range) → AuditSummary
```

### Filters

| Filter | Type | Required | Description |
|--------|------|----------|-------------|
| time_range | [timestamp, timestamp] | Yes | Events within this window |
| event_types | string[] | No | Specific event types |
| entity_id | UUID | No | Events involving this entity |
| actor_id | UUID | No | Events performed by this actor |
| severity | enum | No | info, warning, critical |
| tags | string[] | No | Audit tags to match |
| correlation_id | UUID | No | Events in a correlation chain |

### Audit Record

```
AuditRecord {
  event_id: UUID,
  event_type: string,
  timestamp: HLC,
  actor: EntityInfo,        // who did it
  target: EntityInfo,       // who it was done to
  action: string,           // what was done
  result: string,           // success, denied, failed
  severity: string,         // info, warning, critical
  details: JSONB,           // event-specific data
  correlation_id: UUID?,
  chain: EventID[]          // related event chain
}
```

### Query Execution

```
1. Parse and validate filter
2. Check requesting entity has access to requested scope
3. Forward query to EVS with audit tag filter
4. Apply server-side filters (time range, event types)
5. Apply access controls (filter out unauthorized Events)
6. Return paginated results
7. Log AuditQueryExecuted Event
```

## Audit Retention

| Retention Tier | Duration | Storage | Query Latency |
|---------------|----------|---------|---------------|
| **hot** | 90 days | Active Event Store (SSD) | <100ms |
| **warm** | 1 year | Fast-access archive (HDD) | <1s |
| **cold** | 7 years | Compressed archival storage | <60s |
| **permanent** | Indefinite | Constitutional Events only | <60s |

### Archival Process

```
1. Retention policy check runs daily
2. Events past hot retention moved to warm (EVS tier migration)
3. Events past warm retention compressed and moved to cold
4. Cold storage indexed for search
5. Permanent Events never move from cold
6. Archival produces AUS.AuditRetentionTriggered Event
```

## Authorization

Audit access is tiered by role:

| Role | Scope | Query | Export |
|------|-------|-------|--------|
| Security Council | Full system | All Events | All formats |
| Organization Manager | Org-scoped | Org Events | CSV, JSON |
| Entity Supervisor | Entity-scoped | Entity Events | JSON |
| Entity Self | Self only | Own Events | None |
| External Auditor | Granted scope | Granted Events | Granted format |

Authorization check flow:

```
1. Extract requesting entity role from auth token
2. Determine scope from role
3. Apply scope to query (add scope filter)
4. If scope is insufficient for requested query, deny
5. Log authorization decision
```

## Export

### Supported Formats

| Format | Description | Use Case |
|--------|-------------|----------|
| **JSON** | Line-delimited JSON objects | Machine processing |
| **CSV** | Comma-separated values | Spreadsheet analysis |
| **JSONL** | JSON lines | Streaming large exports |
| **Parquet** | Columnar format | Analytics |

### Export Process

```
1. Request export with format and filter
2. Validate authorization for export
3. Create export job (asynchronous for large exports)
4. Execute query against EVS
5. Transform to requested format
6. Write to export storage
7. Notify requester when ready
8. Export available for 7 days, then deleted
```

## Cross-Cutting Concerns

### Security

AUS access is strictly authorized. Unauthorized audit queries are denied and logged as security Events. Export controls prevent bulk data exfiltration. AUS authenticates to EVS for read operations.

### Evidence

Every audit query produces an AuditQueryExecuted Event. Audit exports are tracked with export Events. Retention actions are recorded. AUS itself is auditable — all AUS operations produce Events.

### Lifecycle

AUS follows Platform service lifecycle. Audit retention policies follow their own lifecycle: Defined → Active → Reviewing → Updated. Retention tier transitions are lifecycle events.

### Capability Bounds

AUS is read-only over the Event Store. It cannot modify or delete Events. It cannot bypass EVS to access storage directly. It has no authority over entity state or system configuration.

### Communication

AUS communicates through ACF. Audit queries arrive via ACF messages. Query results are returned through ACF response streams. Export files are delivered through ACF streaming.

### Design DNA

| Rule | Assessment |
|------|-----------|
| R1 — Modulsingularity | AUS does one thing: queryable audit trail |
| R2 — Dependency Order | AUS depends on EVS and ACF; no upward deps |
| R3 — DRY | Audit events are stored once in EVS; AUS indexes |
| R4 — Builder Pattern | Audit queries are built by QueryBuilders |
| R5 — Liskov | All audit storage tiers implement same AuditStore interface |
| R6 — DI over Singletons | AUS receives EVS and ACF as injected dependencies |
| R7 — Tests Exist | Every query filter, export format, and auth scope has tests |
| R8 — Tests Fast | Audit queries complete in <100ms for hot data |
| R9 — Deterministic | Same query + same data always produces same results |
| R10 — Simpler Over Complex | Audit queries are filtered scans; no complex joins |
| R11 — Refactor Over Rewrite | Query schemas evolve via versioned updates |
| R12 — Embrace Errors | Every query failure has a unique error code |
| R13 — Design for Failure | AUS fails closed; unavailable = deny query |
| R14 — Paved Path | AUS is the only path for audit queries |
| R15 — Open/Closed | New audit query types extend without modifying core |
