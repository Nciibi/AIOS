## Purpose

EPG processes Event streams through a directed acyclic graph (DAG) of processors. Each node in the graph performs a specific transformation — filter, transform, enrich, aggregate, or route. EPG is used by Academy (knowledge derivation), AUS (audit filtering), Risk (risk event processing), and Analytics (metric derivation). EPG provides a declarative way to define Event processing pipelines.

## EPG DAG

```
Events → filter → transform → enrich → aggregate → route → output streams
```

### Node Types

| Node Type | Description | Example |
|-----------|-------------|---------|
| **filter** | Include/exclude Events based on predicate | `event_type == "LMS.StateChanged"` |
| **transform** | Modify Event structure | Add derived fields, rename fields |
| **enrich** | Add data from external sources | Look up entity metadata, add context |
| **aggregate** | Combine multiple Events into one | Count Events over window, compute average |
| **route** | Send Events to different output streams | Split by event type, priority, severity |

### Edge Types

| Edge Type | Description |
|-----------|-------------|
| **passthrough** | Event passes to next node unmodified |
| **conditional** | Event passes to next node only if condition met |
| **split** | Event is duplicated to multiple downstream nodes |
| **merge** | Events from multiple upstream nodes converge |

## EPG Operations

```
defineGraph(definition) → graph_id
activateGraph(graph_id) → void
deactivateGraph(graph_id) → void
getGraphStatus(graph_id) → GraphStatus
getProcessorMetrics(graph_id) → ProcessorMetrics[]
updateGraph(graph_id, updates) → void
deleteGraph(graph_id) → void
```

### Graph Definition Schema

```
EPGGraph {
  id: string,
  name: string,
  source_streams: string[],
  nodes: ProcessorNode[],
  edges: Edge[],
  output_streams: string[],
  error_stream: string,
  max_retries: int
}
```

## Fault Tolerance

EPG is fault-tolerant at the node level:

| Fault | Handling |
|-------|----------|
| Node failure | Retry up to 3 attempts with exponential backoff |
| Persistent failure | Event sent to dead letter queue after 3 retries |
| Upstream node failure | Downstream nodes buffer; backpressure applied |
| Graph crash | Graph state is persisted; recovery replays from last checkpoint |

Dead letter queue Events include: original Event, failed node, retry count, error details, and timestamp of first failure.

## EPG Nodes

### Filter Nodes

Define predicates using Event fields. Supported operators: eq, neq, gt, gte, lt, lte, in, not_in, exists, matches (regex), contains.

### Transform Nodes

Apply transformations: add_field, remove_field, rename_field, cast_field, compute_expression. Transformations are deterministic — same input always produces same output.

### Enrich Nodes

Enrichment sources: Entity metadata (from IDS), Service data (via ACF query), External data (via EIP connectors). Enrichment is cached (TTL configurable per source).

### Aggregate Nodes

| Window Type | Description |
|-------------|-------------|
| **tumbling** | Fixed time windows, non-overlapping |
| **sliding** | Fixed time windows, overlapping |
| **session** | Windows based on activity gaps |
| **count** | Fixed count of Events |

### Route Nodes

Routes map Event properties to output streams. Default route: unmatched Events go to error stream.

## EPG Events

| Event Type | Produced When | Fields |
|-----------|--------------|--------|
| `EPG.GraphDefined` | A new processing graph is defined | graph_id, node_count, source_streams, output_streams |
| `EPG.GraphActivated` | A graph begins processing | graph_id, activated_at |
| `EPG.GraphDeactivated` | A graph stops processing | graph_id, deactivated_at, reason |
| `EPG.NodeProcessed` | A node processes an Event | graph_id, node_id, event_id, result, duration_ms |
| `EPG.NodeFailed` | A node fails to process an Event | graph_id, node_id, event_id, retry_count, error |
| `EPG.EventDeadLettered` | An Event exceeds retry limit | graph_id, node_id, event_id, original_error |

## Cross-Cutting Concerns

### Security

Graph definitions are protected. Only authorized entities (Academy, AUS, Risk, Analytics) may define graphs. Graph activation requires Security Council authorization. Processing nodes cannot access unauthorized data.

### Evidence

Every Event processed through EPG produces a NodeProcessed Event. Failures produce NodeFailed Events. Dead letter Events are preserved for audit. Graph lifecycle Events are recorded.

### Lifecycle

EPG graphs follow a lifecycle: Draft → Active → Paused → Updating → Deactivated → Archived. Graph definitions are versioned. Active graphs can be updated without stopping processing (hot reload).

### Capability Bounds

EPG only processes Events. It does not store Events long-term (EVS does). It does not interpret Event meaning (consumers do). It has no authority over source systems. EPG capabilities are limited to DAG-based Event transformation.

### Communication

EPG consumes Event streams from ACF subscriptions. Processed Events are published to ACF output streams. EPG communicates with enrichment sources through ACF queries.

### Design DNA

| Rule | Assessment |
|------|-----------|
| R1 — Modulsingularity | EPG does one thing: DAG-based Event processing |
| R2 — Dependency Order | EPG depends on ACF and EVS; no upward deps |
| R3 — DRY | Graph definitions are the single source for processing logic |
| R4 — Builder Pattern | Graphs are built by GraphBuilders with validation |
| R5 — Liskov | All processor nodes implement the same Processor interface |
| R6 — DI over Singletons | EPG receives ACF as injected dependency |
| R7 — Tests Exist | Every node type and graph configuration has tests |
| R8 — Tests Fast | Node processing tests complete in <5ms |
| R9 — Deterministic | Same Event + same node always produces same output |
| R10 — Simpler Over Complex | DAG is explicit; no hidden control flow |
| R11 — Refactor Over Rewrite | Graph definitions evolve via versioned updates |
| R12 — Embrace Errors | Every node failure has a unique error code |
| R13 — Design for Failure | Retry logic, dead letter queue, checkpoint recovery |
| R14 — Paved Path | EPG is the only path for Event stream processing |
| R15 — Open/Closed | New node types extend without modifying EPG core |
