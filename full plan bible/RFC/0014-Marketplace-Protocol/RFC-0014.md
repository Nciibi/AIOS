## Problem Statement

There is currently no mechanism for distributing, discovering, or installing system extensions (plugins, templates, capabilities) from external sources. Each extension must be manually deployed by the Security Council.

Without a marketplace:
- Plugin distribution requires manual file transfers.
- No version management or dependency resolution.
- No discovery mechanism — users cannot find available extensions.
- No trust/reputation system for extension quality.

**Affected**: Plugin SDK (RFC-0013), External developers, System administrators.

**Evidence**: `Research/003-Phase-5-Ecosystem.md` (Area 2: Marketplace & Distributed Plugin Registry). Plugin SDK (RFC-0013) defines how to build plugins but not how to distribute them.

## Proposed Solution

Introduce the **Marketplace Protocol** — a distributed registry and distribution system for plugins, templates, and capability modules, with version management, dependency resolution, and trust verification.

### Architecture

```
Marketplace (Distributed Registry)
  │
  ├── Public Registry (default)
  │   ├── Discoverable by all AIOS instances
  │   ├── Community-contributed plugins
  │   └── Certification tiers: verified, reviewed, certified
  │
  ├── Private Registry (org-specific)
  │   ├── Internal plugins, proprietary templates
  │   ├── Access-controlled
  │   └── Org-specific distribution
  │
  └── Local Registry (instance cache)
      ├── Installed plugins
      ├── Cached metadata
      └── Offline availability
```

### Registry Entry Schema

```yaml
marketplace_entry:
  plugin_id: "com.example.my-plugin"
  name: "My Plugin"
  version: "1.0.0"
  type: worker_template | capability_module | runtime_adapter | connector

  author:
    name: string
    id: UUID                               # Registry identity

  description: string
  tags: [string]
  categories: [string]

  distribution:
    package_url: string                    # Signed package URL
    package_hash: string                   # SHA-256
    package_format: tar.gz | zip
    package_size_bytes: integer

  compatibility:
    min_system_version: string
    max_system_version: string
    platforms: [string]                    # OS/arch combinations

  dependencies:
    - plugin_id: string
      version_constraint: string           # Semver range

  certification:
    level: verified | reviewed | certified
    certified_by: string
    certification_date: timestamp
    certification_expiry: timestamp

  metrics:
    download_count: integer
    install_count: integer
    average_rating: float
    review_count: integer
    last_updated: timestamp
```

### Distribution Protocol

```yaml
marketplace_distribution:
  # Package format
  package:
    format: signed_tar_gz
    components:
      - manifest.yaml                      # Plugin manifest
      - plugin.wasm | plugin.js | plugin.py# Executable
      - assets/                            # Templates, prompts, configs
      - signature.sig                      # Author signature
      - checksums.txt                     # File-level hashes

  # Signing
  signing:
    algorithm: Ed25519 | ECDSA-P256
    author_key: string                     # Public key fingerprint
    registry_key: string                   # Registry endorsement
    required_verifications:
      author_signature: required
      registry_endorsement: recommended

  # Delivery
  delivery:
    protocol: https
    cdn_enabled: true
    cache_ttl_seconds: 3600
    progressive_download: true             # Download on first use, not install
    delta_updates: true                    # Download only changed files
```

### Search & Discovery

```yaml
marketplace_search:
  query:
    text: string
    type: [worker_template, capability_module, runtime_adapter, connector]
    categories: [string]
    tags: [string]
    certification_level: [verified, reviewed, certified]
    author: string
    sort_by: relevance | downloads | rating | updated
    limit: integer
    offset: integer

  response:
    total_results: integer
    results:
      - plugin_id: string
        name: string
        version: string
        author: string
        description: string
        type: string
        certification: string
        rating: float
        download_count: integer
        tags: [string]
        updated_at: timestamp
```

### Install Flow

```
Sou / Security Council
  │
  ├── 1. Search marketplace for "rust-testing-plugin"
  ├── 2. Select plugin
  ├── 3. Review: certification, dependencies, permissions
  ├── 4. Approve installation
  │
  ▼
Marketplace Client
  ├── 5. Resolve dependencies (recursive)
  ├── 6. Verify signatures
  ├── 7. Download package(s)
  ├── 8. Verify hashes
  ├── 9. Extract to plugin directory
  ├── 10. Run pre_install hooks
  ├── 11. Register with Plugin Runtime (RFC-0013)
  ├── 12. Run post_install hooks
  └── 13. Report installation to marketplace
```

### Version Management

```yaml
marketplace_versioning:
  version_format: semver                    # MAJOR.MINOR.PATCH
  version_constraints:
    exact: "1.0.0"
    caret: "^1.0.0"                        # >=1.0.0, <2.0.0
    tilde: "~1.0.0"                        # >=1.0.0, <1.1.0
    range: ">=1.0.0 <2.0.0"
    any: "*"

  update_policy:
    major: manual_approval
    minor: auto_update
    patch: auto_update
    check_interval_hours: 24
    auto_update_window: "02:00-04:00"       # Off-peak hours
    rollback_on_failure: true
    max_rollback_versions: 3

  conflict_resolution:
    policy: fail_on_conflict | prefer_newest | prefer_installed
    dependency_sharing: allow_duplicates | share_single
```

### Registry Federation

```yaml
marketplace_federation:
  # Registry types
  registries:
    - id: "public"
      url: "https://marketplace.aios.io"
      type: public
      trust_level: high
      required: true for public plugins

    - id: "org-acme-private"
      url: "https://marketplace.acme.com/aios"
      type: private
      trust_level: maximum
      auth_required: true
      scope: acme-corp

    - id: "local"
      type: local
      path: "/var/aios/plugins/cache"

  # Federation protocol
  federation:
    search_scope: all_registries | specific_registries
    mirror_policy: cache_metadata | cache_full | no_cache
    sync_interval_hours: 12
```

### Alternatives Considered

1. **No marketplace (current approach)**: Rejected — manual distribution does not scale.

2. **Centralized registry only**: Rejected — does not support private/org-specific plugins.

3. **Package manager analogy (npm, pip)**: Accepted. Marketplace Protocol follows established patterns with security hardening for the AIOS context.

## Constitutional Review

### Law Compliance

- Does this RFC violate any Law? **No** — Marketplace is a distribution mechanism. Installed plugins operate within capability bounds (Law 4).
- Does this RFC require a Law amendment? **No**
- Does this RFC require a Physics change? **No**

### Design DNA Compliance

| Rule | Assessment |
|------|------------|
| R1 — Modulsingularity | Marketplace handles distribution only. Plugin Runtime handles execution. |
| R2 — Dependency Order | Marketplace depends on Plugin SDK (package format). Plugin Runtime depends on Marketplace for installation. |
| R3 — DRY | Distribution logic centralized in Marketplace rather than duplicated per plugin. |
| R7 — Tests Exist | Package verification tests, dependency resolution tests, federation sync tests, rollback tests. |
| R10 — Simpler Over Complex | HTTP-based registry. Standard package format (tar.gz). No blockchain, no decentralized storage (v1). |
| R12 — Embrace Errors | Marketplace errors: MKT_001 (package not found), MKT_002 (signature invalid), MKT_003 (dependency conflict), MKT_004 (registry unreachable). |
| R13 — Design for Failure | Local cache provides offline availability. If registry is unreachable, installed plugins continue working. |

## Migration Plan

1. **Phase 1** (registry): Deploy public Marketplace Registry.
2. **Phase 2** (client): Marketplace Client in AIOS for install/update/search.
3. **Phase 3** (publishing): Enable plugin authors to publish to marketplace.
4. **Phase 4** (private): Support private registries for org-specific plugins.
